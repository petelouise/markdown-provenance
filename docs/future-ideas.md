# Future Ideas

Ideas deferred from the MVP. Roughly in priority order.

---

## Live editor syntax highlighting

Add CodeMirror 6 `ViewPlugin` + `Decoration` so provenance spans are visually highlighted while you type, not just in Reading mode. This is the natural next step after the MVP is validated.

## Smart selection rewrite

User highlights AI-generated text (`%a{...}`), rewrites it, and the provenance automatically updates to `%s` (self) without ever exposing the raw syntax. Would require CM6 editor integration and careful transaction handling.

## Block markers

Support block-level provenance:
- `%a>` — a single paragraph block (like blockquote syntax)
- `%%%a` / `%%%` — fenced block for multi-paragraph regions

## Named authors

Allow identifying a specific third-party source inline:

```
%q.shakespeare{To be or not to be.}
```

Useful for collaborative notes or annotated bibliographies.

## Attributes / metadata

Record model or tool metadata alongside provenance:

```
%a(model=gpt-4o){This was generated by a specific model.}
```

## User-configurable provenance types

Let users define their own sigils, names, and colors in plugin settings. The hardcoded defaults (`%a %s %q %u`) would remain the baseline.

## `.mdp` file extension

Revisit once the format matures beyond Obsidian. Not recommended right now: Obsidian's graph view, backlinks, search, and sync are all wired to `.md`. The frontmatter `provenance:` key is a cleaner opt-in signal within Obsidian. A custom extension makes more sense if MDP becomes a cross-app standard.
